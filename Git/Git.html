Git 存储在本地电脑上的 分布式管理 版本管理工具 SVN 版本管理工具 集中式管理 GitHub 需要联网 存储在网络上面的 基友平台 git 工作区 > 暂存库 > 版本库 .git同级为工作区 git clone [url] 克隆GitHub上的文件 git config --global user.name "GitHubName" 设置贡献者 git config --global user.email "GitHubEamil" 贡献者邮箱 git config --list 查看配置 查看git的状态 git status 从工作区上传暂存区 指令 git add 文件名 从暂存区上传版本图 指令 git commit -m "版本描述" 从工作区直接到版本库(提交已经被提交过的文件) git commit -a -m "版本描述" 打印关于版本的信息 git log . 修改过的文件 git init 初始化 git diff 工作区和暂存区的差异对比 git diff --cached 暂存区与版本库的差异对比 git diff master 工作区与版本库的差异对比 git reset HEAD 从暂存区撤销 git checkout -- 版本库里面替换掉工作区里面的文件 git rm -f 删除(如果工作区和暂存区都有的话，全部进行删除) git rm --cached (如果工作区和暂存区都有的话，只删除暂存区，保留工作区) git checkout commit_id 将误删的工作区的文件进行恢复 (commit_id 不用全部写全，截取一部分就可以) git commit --amend 将上次提交的内容与本次提交的内容合并 合并格式： git commit -m "版本描述" --amend 处于工作区红色 处于暂存区绿色 同步远程仓库 git remote 查看远程仓库的名字 git remote -v 仓库名字对应的地址 git push 添加到github上 origin是自己定的 ster 分支
